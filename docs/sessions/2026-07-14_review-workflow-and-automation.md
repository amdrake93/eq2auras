## Why-chains worth remembering

- **The EQ2 parser is the swing stream's author, and now fully known.** 14 compiled regexes, first-match-wins, emitting only SwingTypes {1,2,3,10,13,16,20}. Load-bearing surprises: **wards/intercepts arrive as `Healing` swings** (DamageType `Absorption`/`Interception`) and the parser reconstructs true hit values by adding a same-second absorb back onto the following "no damage" hit; **ACT's −2..−5 fail sentinels are never emitted** — only true miss is −1, everything else (parry/riposte/block/dodge/resist) is `Dnum(−9, "<why>")`; threat positions construct `Dnum.ThreatPosition` which **self-clamps to −9/Unknown** (never key threat detection on it); the plugin runs its own `SetupEQ2EnglishEnvironment()` at load that `Clear()`s the shared ColumnDefs/ExportVariables — a **registration-wipe hazard** for any plugin (us) that registers metrics. Full detail: `docs/act-parse-engine.md` §The EQ2 parsing plugin.
- **A finding names the defect, not the remedy.** The load-bearing review-process principle, discovered by watching the reviewer overstep. The reviewer is *deliberately* denied the writer's context, so it's the party least equipped to author a fix — and when it prescribes one, the re-review checks "did the writer do what I said?" instead of "is the defect gone?" (the anchor bias). The fix removes the source rather than managing it: reviewer flags, writer resolves. This **hid for rounds precisely because the artifact under review was a review workflow** — a reviewer proposing process changes read as review content, not overstep. It'd be obvious in any spec/plan/code review.
- **A fresh channel catches bias a tainted one can't.** A closure round's defining move is a *fresh holistic pass*; a session steeped in the back-and-forth is structurally unable to be fresh. Proven twice: the incumbent reviewer flagged the gate-ownership contradiction once and left it propagated to sibling restatements; the dogfood's cold reader (zero conversational context) swept the whole class. The incumbent also once caught *itself* inferring "both flows missed it" without reading the audit files — the "verify, don't trust" rule turned reflexively on its own claim.
- **The re-review verifies the artifact, not the reviewer's own proposal.** Corollary of defect-not-remedy: "is the defect gone, judged fresh against the tree?" not "was my suggested remedy applied?"
- **Commit-per-finding was relaxed to writer's-call** (owner decision) — re-reviews verify the fix *range*, not per commit, so granularity carries no review weight; keep history iterable, keep every fix named.

## Process evolution (this session)

- **The review workflow is now a committed contract** (`docs/review-workflow.md`, pointed to from CLAUDE.md), covering six review types, the verification standard, the feedback-block format, decision routing, the writer's side, per-role bootstrap, and the automated variant. It supersedes the scattered reviewer-workflow memories in spirit — treat the doc as canonical.
- **Automated in-session review is a real capability now.** Owner-invoked per artifact (never writer-initiated); fresh isolated subagent, fixed trigger prompt, no writer-conversation leakage; same-agent continuation for re-reviews (fresh + prior block if lost); bidirectional verbatim surfacing + reviewer-authored audit files outside both git trees; loop runs to closure (audit-trail, not pause-gate); three break-to-owner conditions (Question findings, evidence-backed deadlock, round cap 5). The dogfood ran two rounds fully autonomously (~100k tokens) to a clean closure and materially improved the doc.
- **Two review channels are complementary, not redundant.** Alex's separate independent session and the automated dogfood found *different* real defects on the same artifact. For high-stakes work, running both is defensible; the doc preserves the option explicitly.
- The doc was reviewed under the exact process it defines — including a request-changes round, an owner decision recorded mid-flight, and a self-correcting closure. As tested as a process contract gets.
