## Process evolution (the meta-story)

1. **Branches for development** (2026-07-02): review via `git diff main..branch`, merge on Alex's explicit approval; doc-only tweaks may go straight to main.
2. **Branch verify-CI**: every push builds tests + WPF compile + artifact; publish/stamp are main-only. Caught two compile errors pre-release. The branch artifact doubles as the manual pre-merge test hatch.
3. **Third-party reviewer agent**: separate Claude session reviews specs/plans, pasted in as self-contained blocks with a receiving-code-review directive. Pattern: verify every claim against the working tree; "plan-watch items" carry via the backlog into the next plan, where the plan review checks them. It cannot see this conversation — decisions that resolve to "no change" leave no repo trace, so record them or tell it.
4. **Fix flow** (field-driven spec+code, no plan doc) — legitimate for reversals/small reworks, but **still pauses for spec review before the merge gate** (Alex, after the grid redesign). Proven necessity: every fast fix left stragglers the review caught.
5. **Discussion ≠ approval** (the one real process failure): affirming comments mid-discussion got treated as a green light; spec edits flew while Alex "hadn't even read anything." Corrected to: no file edits in discussion until an explicit write-it-up; one artifact per turn.
6. Backlog entries are the ledger: shipped notes carry field verdicts verbatim; NEXT UP carries reviewer plan-watch items; seeds are direction, not authorization.

## Technical truths earned this window (supplementing docs/plans/2026-07-01-spike-findings.md)

- **DCJS discipline, extended:** enum knobs 0-default; *positions/sizes nullable* (missing numerics materialize as 0 = a real coordinate/legal value); `Normalize` leaves valid values untouched (cross-thread: engine reads per tick on ACT's thread while overlay-thread saves run `ToJson→Normalize`); retired keys are simply ignored — deletion IS the migration (pinned by the `Retired_scale_keys_are_ignored` test).
- **Live-apply patterns:** shared-instance mutation (tab and trackers hold the same `PanelSettings` objects; ACT UI thread ticks and edits — no race) vs per-tick parameter flow (palette) vs rebuild-once (`SetStyle` — the only place retained visuals are dropped; never per tick).
- **`SettingsStore.Update(settings, mutate)`** — one gate around mutate-and-save; writers on ACT's UI thread and the overlay STA thread can't interleave.
- **Win32/WPF:** re-asserting HWND_TOPMOST moves a window to the top of the topmost band (deterministic stacking; click-through windows never activate/raise). Transparent windows need a filled surface to be hittable. `SourceInitialized` fires on first `Show()`; the HWND and EX styles survive Hide/Show. Aliased 1-DIP lines are device-pixel-exact only at 100% scaling.
- **GitHub:** `/releases/latest` excludes prereleases (free stable-channel pointer). `[skip ci]` is honored natively and skipped runs don't consume run numbers. The Actions API's `/jobs` endpoint 500s transiently while runs spin up — poll the run endpoint with tolerance instead of `gh run watch`.
- **FontDialog speaks points; WPF speaks DIPs** (× 96/72) — store DIPs, display points, or the tab lies.
- ACT's `GetTimerFrames()` no-arg returns all panels' timers (measured 07-02); `TimerData.Panel1Display`/`Panel2Display` are the only routing data.
